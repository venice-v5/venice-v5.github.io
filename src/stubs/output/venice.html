Venice is an open-source Micropython runtime for VEX V5 robots.

from venice import Motor, Gearset, Direction
import venice.vasyncio

async def main():
    my_motor = Motor(
        1,
        Direction.FORWARD,
        Gearset.GREEN
    )
    my_motor.set_voltage(10.0)
vasyncio.run(main())

104class RotationSensor:
105    """A rotation sensor plugged into a Smart Port.
106
107    The VEX V5 Rotation Sensor, measures the absolute position, rotation count,
108    and angular velocity of a rotating shaft.
109
110    # Hardware Overview
111
112    The sensor provides absolute rotational position tracking from 0 to 360 with 0.088 accuracy.
113    The sensor is composed of two magnets which utilize the
114    [Hall Effect](https://en.wikipedia.org/wiki/Hall_effect_sensor) to indicate angular
115    position. A chip inside the rotation sensor (a Cortex M0+) then keeps track of the total
116    rotations of the sensor to determine total position traveled.
117
118    Position is reported by VEXos in centidegrees before being converted to a float
119    in the given unit of rotation.
120
121    The absolute angle reading is preserved across power cycles (similar to a potentiometer), while
122    the position count stores the cumulative forward and reverse revolutions relative to program
123    start, however the *position* reading will be reset if the sensor loses power. Angular velocity
124    is measured in degrees per second.
125
126    Like all other Smart devices, VEXos will process sensor updates every 10mS.
127    """

229class AbstractMotor:
230    """A motor plugged into a Smart Port.
231
232    This is an abstract class supporting shared methods for both the 5W (Exp)
233    and 11W motor variants. To create a motor, use the initializers of `ExpMotor`
234    or `V5Motor`, respectively.
235
236    # Overview
237
238    The V5 Smart Motors come in two variants: [an 11W model](https://www.vexrobotics.com/276-4840.html),
239    with interchangeable gear cartridges and [a 5.5W model](https://www.vexrobotics.com/276-4842.html),
240    with a fixed gearing. The 11W motor supports three cartridge options, which will gear the motor
241    down from its base RPM of 3600: a red cartridge providing 100 RPM output, a green cartridge for
242    200 RPM, and a blue cartridge for 600 RPM. The 5.5W motor comes with a non-interchangeable 200
243    RPM gear cartridge.
244
245    Smart Motors feature several integrated sensors, including an encoder for measuring the velocity
246    and position of the motor, a temperature sensor for detecting overheats, and sensors for
247    measuring output voltage, current, and efficiency.
248
249    Communication between a Smart motor and the V5 Brain occur at two different intervals. While
250    the motor communicates with the Brain every 5 milliseconds (and commands can be written to the
251    motor every 5mS), the Brain only reads data from the motor every 10mS. This effectively places
252    the data *write* interval at 5mS and the data *read* interval at 10mS.
253
254    More in-depth specs for the 11W motor can be found [here](https://kb.vex.com/hc/en-us/articles/360060929971-Understanding-V5-Smart-Motors).
255
256    # Current Limitations
257
258    There are some cases where VEXos or the motor itself may decide to limit output current:
259
260    - **Stall Prevention**: The stall current on 11W motors is limited to 2.5A. This limitation
261      eliminates the need for automatic resetting fuses (PTC devices) in the motor, which can
262      disrupt operation. By restricting the stall current to 2.5A, the motor effectively avoids
263      undesirable performance dips and ensures that users do not inadvertently cause stall
264      situations.
265
266    - **Motor Count**: Robots that use 8 or fewer 11W motors will have the aforementioned current limit
267      of 2.5A set for each motor. Robots using more than 8 motors, will have a lower default current limit
268      per-motor than 2.5A. This limit is determined in VEXos by a calculation accounting for the number of
269      motors plugged in, and the user's manually set current limits using `AbstractMotor.set_current_limit`. For
270      more information regarding the current limiting behavior of VEXos, see [this forum post](https://www.vexforum.com/t/how-does-the-decreased-current-affect-the-robot-when-using-more-than-8-motors/72650/4).
271
272    - **Temperature Management**: Motors have an onboard sensor for measuring internal temperature.
273      If the motor determines that it is overheating, it will throttle its output current and warn
274      the user.
275
276    # Motor Control
277
278    Each motor contains a sophisticated control system built around a Cortex M0+ microcontroller.
279    The microcontroller continuously monitors position, speed, direction, voltage, current, and
280    temperature through integrated sensors.
281
282    The onboard motor firmware implements a set of pre-tuned PID (Proportional-Integral-Derivative)
283    controllers operating on a 10-millisecond cycle for position and velocity control. Motors also
284    have braking functionality for holding a specific position under load.
285    """

Hardware Overview

The sensor provides absolute rotational position tracking from 0 to 360 with 0.088 accuracy. The sensor is composed of two magnets which utilize the Hall Effect to indicate angular position. A chip inside the rotation sensor (a Cortex M0+) then keeps track of the total rotations of the sensor to determine total position traveled.

A motor plugged into a Smart Port.

This is an abstract class supporting shared methods for both the 5W (Exp) and 11W motor variants. To create a motor, use the initializers of ExpMotor or V5Motor, respectively.

Overview

The V5 Smart Motors come in two variants: an 11W model, with interchangeable gear cartridges and a 5.5W model, with a fixed gearing. The 11W motor supports three cartridge options, which will gear the motor down from its base RPM of 3600: a red cartridge providing 100 RPM output, a green cartridge for 200 RPM, and a blue cartridge for 600 RPM. The 5.5W motor comes with a non-interchangeable 200 RPM gear cartridge.

Smart Motors feature several integrated sensors, including an encoder for measuring the velocity and position of the motor, a temperature sensor for detecting overheats, and sensors for measuring output voltage, current, and efficiency.

Communication between a Smart motor and the V5 Brain occur at two different intervals. While the motor communicates with the Brain every 5 milliseconds (and commands can be written to the motor every 5mS), the Brain only reads data from the motor every 10mS. This effectively places the data write interval at 5mS and the data read interval at 10mS.

More in-depth specs for the 11W motor can be found here.

Current Limitations

There are some cases where VEXos or the motor itself may decide to limit output current:

  • Stall Prevention: The stall current on 11W motors is limited to 2.5A. This limitation eliminates the need for automatic resetting fuses (PTC devices) in the motor, which can disrupt operation. By restricting the stall current to 2.5A, the motor effectively avoids undesirable performance dips and ensures that users do not inadvertently cause stall situations.

  • Motor Count: Robots that use 8 or fewer 11W motors will have the aforementioned current limit of 2.5A set for each motor. Robots using more than 8 motors, will have a lower default current limit per-motor than 2.5A. This limit is determined in VEXos by a calculation accounting for the number of motors plugged in, and the user's manually set current limits using AbstractMotor.set_current_limit. For more information regarding the current limiting behavior of VEXos, see this forum post.

  • Temperature Management: Motors have an onboard sensor for measuring internal temperature. If the motor determines that it is overheating, it will throttle its output current and warn the user.

Motor Control

Each motor contains a sophisticated control system built around a Cortex M0+ microcontroller. The microcontroller continuously monitors position, speed, direction, voltage, current, and temperature through integrated sensors.

The onboard motor firmware implements a set of pre-tuned PID (Proportional-Integral-Derivative) controllers operating on a 10-millisecond cycle for position and velocity control. Motors also have braking functionality for holding a specific position under load.
